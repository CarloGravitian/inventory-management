Filter Data Flow
1
User selects a filter
FilterBar.vue updates one of four reactive refs in the useFilters composable: selectedPeriod, selectedLocation, selectedCategory, or selectedStatus.
2
View detects the change
Each view (Dashboard, Inventory, Orders, etc.) watches the shared composable refs. A watch() handler fires and calls getCurrentFilters() to get the current filter state as a plain object.
3
API request is built
api.js converts the filter object into URL query params using URLSearchParams. Values equal to 'all' are omitted. The warehouse filter maps from selectedLocation to the warehouse query param.
4
FastAPI filters in-memory data
apply_filters() does case-insensitive matching on warehouse, category, and status. filter_by_month() handles both direct months (2025-01) and quarter keys (Q1-2025 maps to Jan/Feb/Mar).
5
Response rendered in Vue
Pydantic-validated JSON is returned. Vue views store raw data in ref([]) and expose derived values via computed() properties, which update reactively without re-fetching.

API Endpoints
MethodEndpointFilters SupportedReturns

GET /api/inventory warehouse, category InventoryItem[]
GET /api/inventory/{id} — InventoryItem
GET /api/orders warehouse, category, status, month Order[]
GET /api/orders/{id} — Order
GET /api/demand — DemandForecast[]
GET /api/backlog — BacklogItem[] (+ has_purchase_order flag)
GET /api/spending/summary — Spending totals + YoY % changes
GET /api/spending/monthly — Monthly breakdown by cost type
GET /api/spending/categories — Category amounts + percentages
GET /api/spending/transactions — Transaction[]
GET /api/reports/quarterly — Q1–Q4 stats (orders, revenue, fulfillment rate)
GET /api/reports/monthly-trends — Month-by-month order + revenue trends
POST /api/purchase-orders — PurchaseOrder (created)
Data Files — server/data/
FileContentsKey Fields
inventory.json Stock items across 3 warehouses sku, warehouse, quantity_on_hand, reorder_point, unit_cost
orders.json 12 months of orders (2025) order_number, customer, items[], status, total_value, order_date
demand_forecasts.json Item-level demand projections item_sku, current_demand, forecasted_demand, trend
backlog_items.json Shortage / delayed order items item_sku, quantity_needed, quantity_available, days_delayed, priority
spending.json Procurement, operational, labor, overhead spending_summary, monthly_spending[], category_spending[]
transactions.json Financial transaction log date, description, amount, category, type (debit/credit)
purchase_orders.json POs created from backlog backlog_item_id, supplier_name, quantity, unit_cost, status
